## Контекст

Пользователь предложил альтернативный вариант отображения фото в слоте «в полный рост»: вместо обычного `object-cover` (где у вертикального ростового фото срезается голова или ноги) — показывать **всё фото целиком** (`object-fit: contain`), а пустоты по бокам заполнять **тем же фото в режиме cover с прозрачностью 20%** на подложке `#ACACAC`. Получается «cinema-mode» / blurred-fill эффект, но без блюра — просто полупрозрачный fill.

Это полностью решает проблему кадрирования ростовых фото и хорошо подходит именно для слота `fullHeight`, где важно видеть фигуру целиком.

## Решение

Расширить `PhotoSlot.tsx`, добавив необязательный проп `fitMode: "cover" | "contain-fill"` (по умолчанию `"cover"`).

- `"cover"` (текущее поведение) — для слотов `portrait`, `back`, `profile` (с фиксом `object-position: top` из предыдущего шага).
- `"contain-fill"` — для слота `fullHeight`:
  1. Фоновый слой: `<img>` того же src, `object-fit: cover`, `opacity: 0.2`, на подложке `#ACACAC` (через CSS-переменную `--vf-photo-fill-bg`, чтобы оставалось настраиваемым из конфига).
  2. Передний слой: `<img>` того же src, `object-fit: contain`, по центру.
  3. Подпись-pill и кнопка удаления — поверх обоих слоёв (как сейчас).

В `CreateProfile.tsx` при рендере `PhotoSlot` пробрасывать `fitMode={slot.id === "fullHeight" ? "contain-fill" : "cover"}`. Альтернативно (предпочтительно для соответствия памяти проекта про конфигурируемость) — добавить поле `fitMode` в `PhotoSlotConfig` в `widget/config/types.ts` + `defaults.ts`, чтобы админка могла менять режим для любого слота.

## Что меняется

- `src/widget/config/types.ts` — поле `fitMode?: "cover" | "contain-fill"` в `PhotoSlotConfig`.
- `src/widget/config/defaults.ts` — `fullHeight` получает `fitMode: "contain-fill"`, остальные слоты — `"cover"` (или просто оставить undefined = cover).
- `src/widget/config/types.ts` (тема) и `src/widget/config/defaults.ts` — новая CSS-переменная `--vf-photo-fill-bg` со значением `#ACACAC` (light/dark — можно подобрать темнее для dark-темы).
- `src/components/widget/ui/PhotoSlot.tsx` — поддержка `fitMode`: рендер двух `<img>` для режима `contain-fill`, для `cover` поведение не меняется (только текущий фикс `object-position: top`).
- `src/components/widget/CreateProfile.tsx` — передача `fitMode` из `slot.fitMode` (если выберем конфиг-вариант).

## Риски и проверка

- Изменение аддитивное: для слотов кроме `fullHeight` поведение остаётся прежним.
- Подпись-pill «в полный рост» и крестик удаления остаются на тех же местах поверх изображения — проверить на тёмном fill, что они читаемы (pill уже на белой подложке, крестик — белый круг).
- Проверка: загрузить ростовое фото (вертикальное) → видна вся фигура целиком; по бокам — приглушённое фото на сером фоне, как на референсе.
- Адаптивность: контейнер квадратный, оба `<img>` `absolute inset-0 w-full h-full` — корректно работает на 320–480px и десктопе.
- Эмодзи/хардкоды не вводятся, цвет вынесен в CSS-переменную, режим — в конфиг.

## Открытый вопрос

Применять `contain-fill` **только** к `fullHeight` (как на референсе) или ко всем «фигурным» слотам (`fullHeight` + `back` + `profile`)? Рекомендую только `fullHeight` — для портретных кадров `cover` + `top` смотрится плотнее и ближе к дизайну скрина 2. Если согласны — реализую так.
